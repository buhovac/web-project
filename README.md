# IFOSUP Web Project

Application web développée dans le cadre du cours **IFOSUP – Web Project**.

Ce projet consiste à réaliser une application en ligne respectant une série de critères techniques et de bonnes pratiques, afin de valider progressivement différents degrés de maîtrise.

---

## Objectif général

Réaliser une application web comprenant :

- Une structure claire (templates, navigation, pages).
- Un système de gestion de comptes utilisateurs.
- Des fonctionnalités côté client (JavaScript, bibliothèques).
- Une base de données et une API.
- Des mesures de sécurité côté serveur et côté client.
- Une attention particulière à la cohérence, la précision et l’intégration des concepts.

---

## Palier minimal – 50%
### Fonctionnalités obligatoires

Pour **réussir et atteindre le palier des 50%**, toutes les demandes de la liste ci-dessous doivent être respectées.  
De plus, l’application **ne doit pas rencontrer de bugs bloquants**.

-  Système de template :
    - en-tête
    - contenu
    - pied de page
-  Système de navigation pour accéder aux différentes pages du site web.
-  Page d’inscription permettant de **créer un compte utilisateur**.
-  Page de connexion permettant de **se connecter à son compte utilisateur**.
-  Page de profil affichant au minimum :
    - le **nom** de l’utilisateur,
    - l’**email** de l’utilisateur.
-  Base de données pour **gérer les comptes utilisateurs**.
-  Au moins **deux fonctionnalités côté client** :
    - une développée **from scratch** (JavaScript vanilla),
    - une à l’aide d’une **bibliothèque** (ex. : menu hamburger, carrousel, parallaxe, etc.).
-  Communication avec **une API interne ou externe** :
    - soit au sein du projet progressif,
    - soit via les exercices dédiés.
-  Création d’**au moins une classe** (en **JavaScript** ou en **PHP**) et exploitation concrète de celle-ci dans le projet.

> ⚠️ Si toutes ces conditions ne sont pas remplies, ou si des bugs bloquants sont présents, le résultat est **Échec** ou **50% maximum**.

---

## Degré de Maîtrise 1-2 – Améliorations

Pour aller au-delà du simple palier de réussite, il est demandé d’ajouter les caractéristiques suivantes :

-  **Hébergement en ligne** de l’application.
-  **Code source disponible sur GitHub** (ou autre forge, mais GitHub recommandé).
-  Application **responsive** (adaptée aux mobiles, tablettes et desktops).
-  **Soumission des formulaires avec `fetch`** (AJAX / API fetch).
-  **Mesures de sécurité minimales** :
    - échappement des caractères,
    - configuration correcte du **cookie de session**,
    - utilisation de **requêtes préparées** pour l’accès à la base de données.
- ️ **Mesures de sécurité supplémentaires** :
    - mise en place d’un **jeton CSRF**,
    - **limitation de la fréquence des requêtes** (rate limiting).

---

## Degré de Maîtrise 2-2 – Bonnes pratiques

À ce niveau, une attention particulière doit être portée aux trois points suivants :

1. **Niveau de cohérence**  
   Capacité à établir une majorité de liens logiques pour former un **ensemble organisé** :
    - navigation claire et intuitive,
    - structure de code lisible,
    - logique fonctionnelle cohérente entre les différentes parties du site.

2. **Niveau de précision**  
   Clarté, concision et rigueur au niveau :
    - de la **terminologie**,
    - des **concepts**,
    - des **techniques / principes / modèles** utilisés.

   Le code, les commentaires, la documentation et les messages affichés à l’utilisateur doivent être précis et compréhensibles.

3. **Niveau d’intégration**  
   Capacité à **s’approprier** les notions, concepts, techniques et démarches en les intégrant dans :
    - l’analyse,
    - l’argumentation,
    - la pratique,
    - la recherche de solutions.

   Autrement dit, il ne s’agit pas seulement d’appliquer des recettes, mais de montrer une **compréhension réelle** des choix techniques effectués.

## Résumé des attentes

- **50%** :  
  Toutes les fonctionnalités minimales sont présentes, sans bugs bloquants.

- **Degré de maîtrise 1-2 (Améliorations)** :

- **Degré de maîtrise 2-2 (Bonnes pratiques)** :  
  Projet cohérent, précis, techniquement maîtrisé, avec une bonne intégration des notions vues au cours.
